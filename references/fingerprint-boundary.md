# 指纹能力边界

## 可以做什么

- 读取用户提供的指纹图片
- 判断图片质量是否足以分析
- 基于方向场和奇点检测，尝试做基础纹型分类：
  - loop
  - whorl
  - arch
- 在图像不足时输出 `manual_review_required`
- 给出 `confidence / detected_cores / detected_deltas`
- 为未来的叶束分类模块保留输入

## 不能做什么

- 不能仅凭一张指纹图片判断“对应哪张纳迪叶”
- 不能宣称完成了真实 Agastya / Bhrigu 叶片匹配
- 不能把指纹图像分析结果包装成权威纳迪阅读

## 产品表达原则

- 指纹模块命名为：
  - `指纹探索性分类`
  - `探索性指纹分型`
  - 或 `Thumb Pattern Classification`
- 不命名为：
  - `叶片匹配`
  - `真实纳迪认证`
  - `Palm Leaf Match`

## 何时才可以升级

只有在同时具备以下条件后，指纹模块才有机会进入真实匹配层：

- 明确的手印分类体系
- 与叶束相连的索引表
- 可验证的数据集
- 人工复核流程
