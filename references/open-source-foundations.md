# 开源基础选型

## 目标

明确这个项目应当复用哪些开源基础，而不是从零手写全部吠陀计算。

## 第一优先级

### PyJHora

- 类型：Python 吠陀占星引擎
- 用途：本地底层计算
- 优点：
  - 覆盖广
  - Python 生态友好
  - 测试体量大
- 结论：最适合作为 v0 默认底座

## 第二优先级

### jyotish-api

- 类型：REST API
- 用途：服务化替代
- 结论：适合未来做远程服务，不是当前本地 MVP 首选

### jyotish-mcp

- 类型：API / MCP 风格引擎
- 用途：参考 tool 化与 agent 集成方式
- 结论：对产品形态有参考价值

## 第三优先级

### ndastro-api

- 类型：Nadi 主题 API 壳
- 用途：参考知识组织和内容结构
- 结论：可借知识层，不适合作为唯一计算底层

### BhriguWelt

- 类型：占星产品壳
- 用途：参考知识组织、产品包装与离线优先思路
- 结论：适合作为结构参考，不适合作为真实纳迪叶层

## 明确不用来当底层的

- 宣传网站
- 极小型 skill 仓库
- synthetic simulation 项目

## 选型结论

v0 默认路线：

1. `PyJHora` 负责计算
2. 本工作区自建 adapter
3. 本工作区自建纳迪解释层
